### 1. **wxfile 清理逻辑的两个边界 Bug**
**现象**: `_prune_wxfile_history()` 会误删 img 子目录内不符合日期格式的文件

**根因**:
```python
all_files.extend([p for p in img_dir.glob("*") if p.is_file()])
date_re.match(p.name)  # 对每个文件匹配日期前缀
# → img 下 Markdown 嵌入的图片通常命名格式不同
```

**风险**: 若 `img/` 目录内有旧文件（如 `cover.jpg`, `icon.png`）且不匹配 `YYYY-MM-DD_*` 格式，它们会被保留；若有同名但前缀符合的文件，会被正确清理。

**现象级别**: ⚠️ 低 (基本不影响功能，但留下垃圾)

**修复**:
```python
# 分离清理逻辑：img 子目录额外要求前缀匹配
def _prune_wxfile_history(base_dir, keep_days):
    # base_dir 下的 .md/.html 按日期前缀清理
    base_files = [p for p in base_dir.glob("*") if p.is_file() and date_re.match(p.name)]
    # img 下也按日期前缀清理
    img_files = [p for p in (base_dir / "img").glob("*") if p.is_file() and date_re.match(p.name)]
    for f in base_files + img_files:
        if date_re.match(f.name).group(1) not in keep_dates:
            f.unlink(missing_ok=True)
```

### 4. **Sentiment history 清理 vs wxfile 清理 策略不一致**
**现象**:
```python
_CSV_KEEP_DAYS = 5          # CSV 保留 5 天
_WXFILE_KEEP_DAYS = 5       # wxfile 也保留 5 天
SENTIMENT_HISTORY_MAX_ENTRIES = 200  # sentiment 按条数，不按天数
```

**问题**:
- wxfile 按"日期"计数（最新 5 个交易日）
- CSV 按"日期"计数（最新 5 天）
- sentiment 按"条数"计数（最多 200 条历史）
  - 如果每天有 20 条 sentiment 记录，200 条 ≈ 10 天
  - 如果某周行情停滞，只有 5 条 sentiment，则 200 条 ≈ 40 天

**建议**:
```python
# 统一改成按"天数"
SENTIMENT_HISTORY_MAX_DAYS: int = 20  # 最多保留最近 20 天
# 在 _save_sentiment_history() 改用日期过滤而非截断
```

## 🟡 低优先问题（技术债、维护建议）

### 6. **Wikimedia Commons API 超时与网络脆弱性**
**现象**: 
```python
with urlrequest.urlopen(req, timeout=8) as resp:  # 8秒超时
```

**问题**:
- 8秒对于境内用户访问 Wikimedia 可能太短
- 无重试机制 — 一次网络抖动就失败，走 fallback
- fallback 本身也可能超时（无重试）

**建议**:
```python
# 添加指数退避重试
def _fetch_with_retry(url, max_retries=2):
    for attempt in range(max_retries):
        try:
            return urlrequest.urlopen(url, timeout=8)
        except Exception:
            if attempt < max_retries - 1:
                time.sleep(2 ** attempt)  # 1s, 2s
            else:
                raise
```

### 8. **依赖版本号未锁定**
**现象**: `requirements.txt` 用的是版本范围或无版本号
```
akshare    # 无版本 → 每次 pip install 都可能拉最新，行为不确定

```

**建议**:
```bash

# 这样生产环境版本完全可复现
```
